# Feature Spec Writer

Produce a compact spec that engineering can implement from.

## Workflow

1. Identify the user problem and the desired outcome.
2. Define scope and non-goals.
3. Describe the proposed behavior in plain language.
4. Capture edge cases and operational constraints.
5. End with acceptance criteria that can be tested.

## Output shape

- problem
- goals
- scope
- non-goals
- behavior
- edge cases
- acceptance criteria

## Guardrails

- do not overdesign the implementation unless asked
- separate product requirements from technical suggestions
- write acceptance criteria that are observable
