# Planning

Use this skill when the user has a broad outcome and needs a practical path.

## Workflow

1. Define the outcome in one sentence.
2. Identify constraints, unknowns, and likely blockers.
3. Break the work into phases with visible checkpoints.
4. Mark which assumptions are safe to make now.
5. Keep the plan short enough to execute immediately.

## Output shape

- outcome
- assumptions
- phases
- first concrete action

## Guardrails

- do not create long plans for small tasks
- prefer reversible early steps
- convert planning into execution quickly when the user wants action
