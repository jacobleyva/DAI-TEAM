# DAI Overview

This is a clean-slate DAI workspace for a new technical team.

## Why This Exists

It gives the team a durable, Git-backed operating system for context, projects, knowledge, and reusable work patterns.

## Main Areas

- `core/`: durable rules and standards
- `team/`: collaboration and handoff conventions
- `memory/`: durable shared context
- `projects/`: project entry points and working notes
- `knowledge/`: curated reference material
- `templates/`: reusable note formats
- `members/`: optional person-specific context
- `skills/`: reusable Codex skills
- `automations/`: recurring checks and task definitions
- `tools/`: helper scripts

## Start Here

1. Read `active-work-map.md`.
2. Read `core/`.
3. Read `memory/identity.md` and `memory/current-focus.md`.
4. Use `team/` for collaboration rules.
5. Keep project details in `projects/` and reference material in `knowledge/`.

## Product View

This workspace is not just documentation. It is the coordination layer that makes a team easier to restart, easier to hand off, and easier to scale.
