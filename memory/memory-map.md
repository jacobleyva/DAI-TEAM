# Memory Map

This directory holds durable team context that should persist across sessions.

## Core Memory Files

- [Identity](./identity.md) — lightweight, loaded at session start
- [Preferences](./preferences.md)
- [Current Focus](./current-focus.md) — this week's tactical slice
- [Projects Index](./projects-index.md)
- [Session Context](./session-context.md) — synthesized startup payload (auto-generated; do not edit by hand)
- [Skills Catalog](./skills-catalog.md) — one-line-per-skill index (auto-generated by `tools/scripts/skills_index.sh`)

## EFFECTUS — deeper substrate (read on-demand)

The EFFECTUS folder holds the team's substrate of intent — read implicitly via session-context for identity/current-focus, and explicitly when leverage matters (goal review, big decisions, when something hurts).

- [EFFECTUS Map](./effectus/effectus-map.md) — index of the seven EFFECTUS files
- [Mission](./effectus/mission.md) — long-arc purpose
- [Goals](./effectus/goals.md) — active dated commitments
- [Beliefs](./effectus/beliefs.md) — worldview and frames
- [Wisdom](./effectus/wisdom.md) — hard-won lessons
- [Challenges](./effectus/challenges.md) — active impediments
- [Strategies](./effectus/strategies.md) — how goals are pursued
- [Models](./effectus/models.md) — mental models the team uses

## Working subfolders

- `decisions/` — dated decision logs
- `learnings/` — captured lessons (raw, before they crystallize into EFFECTUS wisdom)
- `session-summaries/` — per-session handoff notes
- `work/` — per-task WORK notes (the system of record per the Algorithm)
- `security/` — security-domain memory (NEW in v1.0 Tier 4): `threat-landscape.md` (active threats), `auth-patterns.md` (proven authn/authz patterns), `incident-log.md` (durable record of incidents), `dependency-findings.md` (auto-generated by `tools/scripts/dep_audit.py`)
