# First Session Guide

## Read Order

The canonical read order lives in `LAUNCHER.md`. Follow that file. For a first-time visitor, the recommended sequence is:

1. `README.md` — what DAI is, the pitch, the install flow
2. `LAUNCHER.md` — the canonical session-start order (LAUNCHER → WELCOME → START_HERE → dai-overview → active-work-map → core doctrine → memory)

## Then Decide

- what is the current priority
- what should be tracked in memory
- what belongs in a project file
- what should become a reusable skill or template

## Success Criteria

A new session should be able to answer:

- what this workspace is
- what to read first
- where current work lives
- how to preserve context for the next person
